# Beta Technical Track

This track measures **technical beta readiness** for the local engineering
workflow.

## Gate Command

```bash
make beta-tech-check
```

Output:

- `evidence/beta-tech/beta-tech-check-<timestamp>.md`
- command logs in `evidence/beta-tech/logs-<timestamp>/`

## Included Controls

1. `go test ./...`
2. registry provenance verification (`bpfcompat history verify`)
3. hostile-artifact hardening suite
4. extended profile catalog count (`matrices/extended-15.yaml` must list 15)
5. profile catalog URL reachability audit (`15 reachable / 0 unreachable`)
6. OSS validation expectations:
   - `cilium-tracepoint-in-c` report summary is `pass`
   - `bcc-execsnoop` report summary is `fail` with expected required-profile incompatibility signal
7. selector behavior:
   - `scripts/select-artifact-variant.sh` selects `perfbuf-fallback` over `ringbuf-modern` for MVP reports
8. selector end-to-end proof:
   - `make runtime-selector-proof` must produce a report with:
     - selected version `v1.0.0-perfbuf`
     - `history verify failed: 0`
     - fetch source using `file://` `artifact_uri` with explicit local-proof opt-in (`BPFCOMPAT_FETCH_ALLOW_FILE_URI=true`)

## Interpretation

- `ready`: technical beta gate passed.
- `not-ready`: one or more technical controls failed; inspect report and logs for exact failing controls.
