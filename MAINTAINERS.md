# Maintainers

This file records the people entrusted with project decision-making, repository
maintenance, security response, and releases. It is not a list of every
contributor.

## Active maintainers

| Maintainer | Role | Scope |
|---|---|---|
| [Eren Arı](https://github.com/ErenAri) (`@ErenAri`) | Project lead and maintainer | All project areas, releases, and security coordination |

The project currently has a single active maintainer. This is a known bus-factor
risk and a reason to document decisions, automate releases, keep recovery
material secure, and grow additional maintainers through sustained public
contribution.

## Responsibilities

Active maintainers are expected to:

- review and merge changes within their scope;
- preserve reproducible, evidence-backed compatibility claims;
- keep CI, releases, dependencies, and documentation healthy;
- respond to security and Code of Conduct reports appropriately;
- disclose relevant conflicts of interest; and
- help qualified contributors grow into review and maintenance roles.

Maintainers are not required to provide private support or guaranteed response
times unless covered by a separate agreement.

## Appointment, access, and removal

The nomination and removal process is defined in
[GOVERNANCE.md](GOVERNANCE.md). Maintainer status is earned through sustained
project work; employment, donation, or sponsorship is never sufficient.

Repository access follows least privilege. Release credentials, organization
administration, and security-report access may be narrower than general merge
access.

## Emeritus maintainers

There are currently no emeritus maintainers.

Emeritus maintainers are recognized for past service but have no implied merge,
release, or security access.
